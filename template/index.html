    Template Module

    This module defines the core abstractions for the framework, including Task, Config, and Problem.

    reasoning_core.template.Task

    Task Base Class

    Overview

    The Task class is the central abstraction and main entry point for defining a reasoning task. All specific reasoning tasks (e.g., Arithmetics, Logic) inherit from this class. It orchestrates the generation of Problem instances guided by a Config object, handling timeouts, retries, and difficulty scaling.

    Generative Process Flow

    1. Configuration: Config parameters (such as seed and level constraints) determine the boundaries for generation.
    2. Logic Generation: generate() runs custom logic to produce a raw Problem instance with its structural data and ground truth answer.
    3. Prompt Formatting: prompt(metadata) translates the raw problem data into a natural language question string.
    4. Finalization: The task bundles everything into the Problem object, automatically attaching extensive tracking attributes into metadata (tokens, timings, keys).

    Required Subclass Implementations

    To create a new task, you must inherit from Task and implement: - generate(self): Houses the core logic to create a single Problem (returning it without a prompt). - prompt(self, metadata): Returns the string question format. - score_answer(self, answer, entry) (or default_score_answer): Logic to grade a model's string answer.

    Advanced Capabilities

    • Dynamic Difficulty: Supports .set_level(i) via the config to scale task complexity.
    • Balanced Batching: generate_balanced_batch() uses balancing_key() and deduplication_key() to ensure generated datasets are balanced and unique.

    reasoning_core.template.Config dataclass

    Configuration Object

    Overview

    The Config class manages the parameters that control the generation of problems for a Task. It handles standard attributes like seed and size, but is primarily designed to facilitate smooth and structured difficulty scaling through dynamic attribute updates and stochastic rounding.

    Key Features

    • Level Management (level): The level attribute is used to scale the difficulty of a task dynamically. When set_level(i) is called, the config runs its update(c) method i times, incrementally modifying constraint parameters (like graph depth, number of nodes, etc.).
    • Stochastic Rounding: Integer fields (excluding internal metadata like seed) are treated differently under the hood. They store floating-point values and are automatically resolved to integers during generation using stochastic rounding. Example: If a parameter num_items scales to 1.5 at level 2, accessing it during generate() has a 50% chance to yield 1 and a 50% chance to yield 2. This allows for ultra-smooth curriculum learning progressions.

    Attributes

    Parameter Type Default Utility
    c float 1.0 The scaling step multiplier applied during each level progression increment.
    level int 0 The current difficulty level of the configuration.
    seed int None The global random seed for ensuring reproducible generations.
    size int None Optional parameter defining target dataset size (used by batch handlers).

    Subclass Implementation

    When defining a config for a specific task, subclass Config and implement the update(self, c) method to define exactly how parameters increase at each difficulty step.

    Functions

    get_true_value(name)

    Returns the unrounded float value of a stochastic field.

    reasoning_core.template.Problem

    Problem Instance

    Overview

    Represents a single generated instance of a reasoning task. It acts as a standardized container holding all necessary information about a generated question, its expected solution, and the context of its creation.

    Key Attributes

    • prompt (str): The formatted natural language question presented to the reasoning model.
    • answer (Any): The ground truth solution or reference output for the problem.
    • metadata (edict): A rich dictionary of generation details. Typical keys include:
      • _task: Name of the task that generated this problem.
      • _level: The difficulty level used during generation.
      • _time: Time taken to generate the problem instance.
      • _config: A snapshot of the Config used.
      • cot (optional): A generated Chain-of-Thought showing step-by-step reasoning.
    • data (Any, optional): The raw structured data (e.g., graphs, equations) underlying the problem.

    Representation

    The Problem class inherits from Mapping, meaning it can be iterated over and accessed like a standard Python dictionary (e.g., problem["answer"]).
